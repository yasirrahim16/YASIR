## Overview

YASIRDEV is a learning-focused project built to help students understand browser signals, client-side behavior, and safe data handling in web environments.

It is designed for:
- cyber security students
- web developers
- privacy researchers
- classroom demonstrations
- lab-based experimentation
- defensive awareness training

It is **not** meant for unauthorized monitoring, hidden tracking, or any activity that violates consent.

## Mission

The mission of YASIRDEV is simple:

- teach how browser APIs work
- show how device metadata is exposed
- demonstrate how network requests are received and logged
- reinforce ethical boundaries
- promote privacy awareness
- support safe lab experimentation

## Core Principles

- Consent first
- Transparency always
- Educational use only
- No hidden collection
- No deceptive deployment
- No real-world misuse
- No unauthorized data gathering

## Architecture

YASIRDEV is documented as a multi-layer learning stack:

### 1) Presentation Layer
HTML and CSS structure the demo and show how content is rendered in a browser.

### 2) Client Logic Layer
JavaScript is used to explore standard browser capabilities and display results.

### 3) Logging Layer
A simple server-side handler stores or displays demonstration output in a controlled environment.

### 4) Transport Layer
Local testing or controlled lab routing is used to understand request flow.

### 5) Review Layer
Logs, outputs, and observed values are inspected as part of the lesson.

## Guided Walkthrough

1. Open the project in your editor.
2. Read the HTML first.
3. Inspect the JavaScript logic.
4. Check how the server receives requests.
5. Review how output is stored.
6. Compare browser values with server values.
7. Record what changed and why.

## How the Demo Works

The learning flow is usually:

1. A page loads in the browser.
2. The browser exposes standard properties and APIs.
3. The page reads allowed information.
4. The server receives a request or message.
5. The project records the result for study.
6. The learner compares expected versus actual behavior.

## Consent Model

A strong consent model should answer these questions:

- Did the user know what was being collected?
- Did the user agree clearly?
- Was the environment authorized?
- Was the test limited to the lab?
- Was the result disclosed honestly?

If the answer to any of these is no, the activity does not belong in this project.

## Privacy Notes

A privacy-aware learning project should avoid surprise collection, hidden persistence, and misleading prompts.

Good habits:
- disclose what is being observed
- minimize collected data
- avoid storing unnecessary values
- sanitize logs
- delete lab data after the exercise
- treat all output as sensitive

## Safety Notes

This project should never be used to:
- track strangers
- bypass consent
- profile users without permission
- collect data from public targets
- impersonate trusted services
- disguise collection behavior
- distribute deceptive links

## Learning Outcomes

By studying a safe version of YASIRDEV, learners can understand:

- how browser permission prompts work
- how client and server code communicate
- how metadata appears in requests
- how logging pipelines are designed
- how privacy risks emerge
- how to document tools responsibly

## Troubleshooting

### The page does not open
Check that your local server is running and that the files are in the correct directory.

### Nothing appears in the log
Verify the file path, browser permissions, and the server-side handler.

### The demo behaves differently on another device
That is normal. Browsers, operating systems, and permission settings can change the result.

### Output looks incomplete
Confirm that the page loaded fully and that no script errors are present.

### Files are not saving
Check permissions on the output folder and make sure the environment allows writes.

### Browser permissions were denied
Reset the browser permission for the lab page and test again with clear consent.

### The server responds but the UI is blank
Open the browser console and inspect for JavaScript or resource loading errors.

## FAQ

### Is YASIRDEV meant for public targeting?
No. It should remain a consent-based educational lab.

### Is this a stealth or surveillance tool?
It should not be. The README documents it as a transparent learning project.

### Can I use it to learn web security?
Yes, within an authorized environment and with privacy-safe practices.

### Can I share the code with students?
Yes, as long as the class uses it for ethical and defensive learning.

### Should I keep logs forever?
No. Keep only what is needed for the lesson, then delete it.

### Can I adapt the README for my own project?
Yes. The structure is reusable for other educational repositories.
